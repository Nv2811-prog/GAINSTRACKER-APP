# Brand Guidelines

GainsTracker has a distinct identity: a **Batman / Gotham** universe layered over a serious strength-training app (Alfred as coach, the "Alfred Protocol", boss-fight progress, Gotham villains as assets). Brand-facing work must respect it. The source of truth is **`BRAND.md`** in the project root.

## Workflow
1. **Read `BRAND.md` first.** If missing, create it from the existing identity (the Batman imagery in the repo, the Alfred/Gotham naming, the dark gold/red palette, the tactile gym aesthetic).
2. **Produce on-brand output.** Voice, naming, color, and imagery must match the guide.
3. **Keep BRAND.md updated** when the identity evolves.

## What BRAND.md must define
- **Name & tagline** — how "GainsTracker" is written and positioned.
- **Voice & tone** — confident, a little theatrical (Alfred-as-butler-coach), motivating, never cringe or over-jokey. Define do/don't phrasings.
- **Naming system** — feature names follow the Gotham metaphor consistently (e.g. "Alfred Protocol", boss fights, the Cave). Don't mix in generic SaaS naming.
- **Color palette** — Gotham dark base, gold and red accents; exact hex values and where each is allowed.
- **Imagery & icons** — Batman/Gotham character assets already in the repo; rules for when/how to use them; PWA icon set (icon-180/192/512, maskable, apple-touch).
- **Typography** — heading vs body treatment that fits the premium-tactile feel.
- **Don'ts** — no off-brand stock illustrations, no breaking the Gotham theme, no emoji-spam in product copy.

## Rule of thumb
If copy or visuals could belong to any generic fitness app, it's off-brand. Push it back toward the Gotham/Alfred identity until it could only be GainsTracker.
